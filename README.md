# API-тесты для тестового задания 2.1

## Структура проекта task2
- /tests - запускаемые тесты
- /internal/client - клиент для работы с HTTP
- /internal/constants - константы для проекта
- /internal/helpers - хелперы, которые используются для написания тестов
- /internal/managers - менеджер, является слоем между тестом и клиентом
- /internal/models - модели запросов и ответов
- /internal/utils - более абстрактные помощники
- .golangci.yaml - конфигурация линтеров и форматтеров

## Запуск тестов

### Предусловия:
1. Установленный Go версии 1.25.1+

Если Go не установлен, скачать его можно [отсюда](https://go.dev/doc/install) 

### Основные шаги

1. Склонируйте репозиторий:

```bash
    git clone https://github.com/alima1k/internship-qa-avito.git
```
2. Перейдите в корневую папку проекта
```bash
    cd task2
```

3. Установите необходимые зависимости:
```bash
   go get -u -v -t all
```

4. Для запуска всех тестов достаточно выполнить команду в корне проекта:

```bash
  go test -v ./... 
```

9. Отдельный тест можно запустить командой:

```bash
  go test <путь до go-файла с тестом>
```

> **_Windows:_** Все вышеперечисленные команды необходимо выполнять в **powershell** оболочке
>

## Запуск линтера

1. Установить линтер [golangci-lint](https://golangci-lint.run/docs/welcome/install/local/)
2. Запуск линтера (из корня проекта):

```bash
  golangci-lint run
```

## Конфигурация линтера и форматтера (описана в файле .golangci.yaml)

Дефолтные линтеры отключены. Включены только:
 - errcheck(находит проигнорированные ошибки) 
 - govet(логические ошибки кода)
 - gosec(ищет уязвимости безопасности)
 - revive(стилистика кода)
 - unused(ищет неиспользуемые поля)
 - whitespace(ищет ненужные пустые строки)

Причем у revive в правилах прописано, чтобы он не обращал внимания на имена переменных 

Также добавлено 2 форматтера:
- gofmt (стандартный форматтер)
- goimports (очищает и упорядочивает импорты)

В правилах прописано, что ошибки выводятся без ограничений
